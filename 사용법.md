# 태양광 PF NAV 시뮬레이션 사용법

---

## 파일 구조

| 파일 | 역할 |
|------|------|
| `SolarPFModel.py` | 시뮬레이션 환경 (상태, 외생충격, PV엔진, NAV계산, 전이함수) — 모델링 수식들이 들어가는 곳 |
| `SolarPFPolicy.py` | 정책 클래스 (현재 no-op) 및 에피소드 실행기 — 아직 활성화 x, 신경 안 써도 되는 부분 |
| `SolarPFDriver.py` | MC 시뮬레이션 실행 + 시각화 (진입점) — 이 친구를 돌리면 PLOT이 뜰 것임 |

---

## 실행 방법

```bash
python SolarPFDriver.py
```

지금은 50회 정도의 PF 경로들을 랜덤생성한 것이라 보면 됨.
50회 몬테카를로 시뮬레이션을 수행하고 4패널 플롯(NAV, 토큰 가격, PV, 완공확률)과 분배금 그래프를 출력한다.
결과 이미지는 `nav_simulation_result.png`, `distribution_per_token.png`, `monthly_distribution_detail.png`로 저장됨.
SEED가 바뀌지 않는 이상 계속 반복 실험해도 같은 결과 나올 거야.

## 주요 파라미터 조정 (`SolarPFModel.py` 상단)

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `T_CONSTRUCTION` | 12 | 건설기간 (월) |
| `T_OPERATION` | 240 | 운영기간 (월) |
| `T_SIMULATION` | 480 | 전체 시뮬레이션 기간 (월, 40년) |
| `T_ROLLOVER` | 12 | 롤오버 주기 (월, 1년마다) |
| `INITIAL_PF_COUNT` | 1 | 초기 PF 수 |
| `ROLLOVER_PF_COUNT` | 1 | 롤오버 시 편입 PF 수 |
| `CAPITAL_TOTAL` | 20억 | 프로젝트당 총 자금조달 |
| `SENIOR_DEBT` | 8억 | 선순위 대출 (40%) |
| `TOKEN_AMOUNT` | 8억 | 토큰화 대상 (40%) |
| `TOKEN_UNIT_PRICE` | 10,000 | 토큰 1개당 가격 (원) |
| `TOKENS_PER_PF` | 80,000 | PF당 토큰 수 |
| `EQUITY_SHARE` | 1/3 | 지분수익 비율 (≈33.3%) |
| `TOKEN_SHARE` | 2/3 | 토큰수익 비율 (≈66.7%) |
| `P_DEFAULT` | 0.003 | 건설 디폴트 월별 확률 (0.3%) |
| `P_CURTAIL` | 0.003 | 운영 출력제한 월별 확률 (0.3%) |
| `R_DISCOUNT_ANNUAL` | 0.045 | 연 할인율 (사회적 할인율) |
| `LAMBDA_DEFAULT` | 0.15 | 완공 위험률 (hazard rate) |
| `P_COMPLETE_INIT` | 0.95 | 완공확률 초기값 |
| `WINDOW_SIZE` | 6 | 시장 데이터 윈도우 크기 (월) |
| `DEGRADATION_RATE_ANNUAL` | 0.005 | 패널 연간 열화율 (0.5%) |

SMP/REC/발전량 GBM 파라미터는 `SMP_PARAMS`, `REC_PARAMS`, `ENERGY_PARAMS`에서 조정한다.

## 시뮬레이션 횟수 변경 (`SolarPFDriver.py`)

```python
trial_size = 50   # 테스트하고 싶은 만큼 돌려보면 됨. 부도나는 것들이 안 보이면 부도확률을 늘리거나 횟수를 늘려볼 것
```

---

## 롤오버 / 토큰 병합 비활성화

```python
# 롤오버 비활성화 (단일 PF만 시뮬레이션)
model = SolarPFModel(..., no_rollover=True)

# 토큰 병합(Reverse Split) 비활성화
model = SolarPFModel(..., no_merge=True)

# 초기 완공확률 오버라이드
model = SolarPFModel(..., p_complete_override=0.90)
```
