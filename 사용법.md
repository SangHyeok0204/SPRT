# 태양광 PF NAV 시뮬레이션 사용법

---

## 파일 구조

| 파일 | 역할 |
| `SolarPFModel.py` | 시뮬레이션 환경 (상태, 외생충격, PV엔진, NAV계산, 전이함수) | -> 여기는 이제 모델링하는 수식들이 들어가는 곳
| `SolarPFPolicy.py` | 정책 클래스 (현재 no-op) 및 에피소드 실행기 | -> 여기는 아직 활성화 x 신경안써도 되는 부분
| `SolarPFDriver.py` | MC 시뮬레이션 실행 + 시각화 (진입점) | -> 이 친구를 돌리면 PLOT이 뜰 것임

---

## 실행 방법

지금은 50회 정도의 PF의 단일 경로들을 랜덤생성한 것이라 보면 됨.
50회 몬테카를로 시뮬레이션을 수행하고 3패널 플롯(NAV, PV, 완공확률)을 출력한다.
결과 이미지는 `nav_simulation_result.png`으로 저장될 것이고
SEED가 바뀌지 않는 이상 계속 반복 실험해도 같은 결과 나올 거야

---

## 주요 파라미터 조정 (`SolarPFModel.py` 상단)

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `T_CONSTRUCTION` | 730 | 건설기간 (일) |
| `T_OPERATION` | 7300 | 운영기간 (일) |
| `P_DEFAULT` | 0.0001 | 건설 디폴트 일일 확률 |
| `P_CURTAIL` | 0.0001 | 운영 출력제한 일일 확률 |
| `R_DISCOUNT_ANNUAL` | 0.045 | 연 할인율 |
| `LAMBDA_DEFAULT` | 0.005 | 완공 위험률 (hazard rate) |
| `P_COMPLETE_INIT` | 0.7 | 완공확률 초기값 |
| `WINDOW_SIZE` | 30 | 시장 데이터 윈도우 크기 (일) |

SMP/REC/발전량 GBM 파라미터는 `SMP_PARAMS`, `REC_PARAMS`, `ENERGY_PARAMS`에서 조정한다.
