# NAV 시뮬레이션 설계 문서
## 신재생에너지 펀드형 STO (태양광 PF)

---

# 1. 목적

본 문서는 태양광 프로젝트 파이낸싱(PF) 구조 기반 펀드형 STO의 월별 NAV 경로 시뮬레이션을 위한 백본 아키텍처를 정의한다.

시뮬레이션 요구사항:

- 월 단위 타임스텝 사용
- 시뮬레이션 범위:
  - 건설기간 1년 (12개월)
  - 운영기간 20년 (240개월)
- NAV 시계열 생성
- 위험중립(Risk-Neutral) / 현실세계(Real-World) 평가 모드 전환 지원
- 희소 외생 포아송 이벤트 반영
- 마르코프 상태 갱신 구조 준수
- 다중 PF 운영 및 롤오버 편입 지원

본 문서는 최종 가치평가 모델이 아니다.
시뮬레이션 엔진의 백본 구조를 정의하는 문서이다.

---

# 2. 시간 구조

- 타임스텝: 1개월
- 건설기간: 12개월
- 운영기간: 240개월
- 전체 기간: 252개월

---

# 3. 자본구조

## 3.1 단일 프로젝트 자본구조

프로젝트당 자금조달 규모: 20억원

| 구분 | 금액 | 비율 |
|------|------|------|
| 자기자본 (사업 관련 투자자) | 4억원 | 20% |
| 선순위 대출 | 4억원 | 20% |
| 토큰화 대상 | 12억원 | 60% |

## 3.2 다중 PF 운영 구조

- 초기 운영 PF 수: 5개
- 롤오버 시 편입 PF 수: 5개 (동일 규모)
- 총 초기 자금조달 규모: 100억원 (20억 × 5개)

## 3.3 토큰 단위

소액 투자자 접근성을 위해 토큰 1개당 가격을 1만원으로 설정한다.

| 항목 | 값 |
|------|-----|
| 토큰 1개당 가격 | 10,000원 |
| PF당 발행 토큰 수 | 120,000개 (= 12억 ÷ 1만원) |
| 초기 발행 토큰 수 | 600,000개 (= 120,000 × 5개 PF) |
| 초기 토큰 자금조달 총액 | 60억원 |

---

# 4. 수익 분배 구조

순현금흐름 분배:

- 순현금흐름 = 지분수익 + 토큰수익
- 분배 비율: 지분수익 : 토큰수익 = 1 : 3

---

# 5. 모델링 프레임워크

마르코프 상태 전이 구조를 사용한다.

매월 t에서:

State S_t → 외생 충격 → 전이 → S_{t+1}

파생 변수 (PV_t, P_complete_t, NAV_t)는 S_t로부터 계산된다.

---

# 6. 상태 변수

상태 S_t에 포함되어야 할 변수:

## 핵심 상태

- `time_t` : 정수 (현재 월 인덱스)
- `status_t` : 범주형
  - "PRE"  : 완공 전
  - "POST" : 완공 후
  - "FAILED" : 건설 디폴트

## 시장 데이터 윈도우

윈도우 크기: n개월 (기본값 6개월)

- `SMP_t` : shape (n, 2)
  - [가격, 거래량]
- `REC_t` : shape (n, 2)
  - [가격, 거래량]
- `energy_amount_t` : shape (n,)

t-n+1월부터 t월까지의 데이터를 나타낸다.

## 재무 파라미터

- `r_t` : 고정 사회적 할인율
  - 연 4.5% 고정

## 외생 이벤트 플래그

- `default_today` : bool (당월 디폴트 발생 여부)
- `curtail_today` : bool (당월 출력제한 발생 여부)

---

# 7. 외생 이벤트 (포아송 기반)

월별 베르누이 시행 (포아송 근사)을 가정한다.

## 7.1 건설 디폴트 이벤트

- status == PRE 일 때만 적용
- 월별 확률:
  p_default = 0.3% = 0.003

발생 시:
→ status = FAILED
→ NAV = 0 (영구)

---

## 7.2 운영 출력제한 이벤트

- status == POST 일 때만 적용
- 월별 확률:
  p_curtail = 0.3% = 0.003

발생 시:
→ 해당월 SMP 및 REC 수익 = 0
→ PV 계산에 반영

---

# 8. 완공 모델

status == PRE 동안 완공 확률이 매월 갱신된다.

## 8.1 완공 확률 정의

P(t) = h(t) × g(t)

- h(t): 위험률 함수 (hazard rate function)
- g(t): 바이너리 변수 (0 또는 1)

## 8.2 g(t) 확률 과정

- 매 시점 t에서 0이 될 확률: 0.3% (0.003)
- 포아송 분포가 아닌, 독립적 베르누이 시행
- g(t) = 0 발생 시 → 프로젝트 실패 (FAILED)

## 8.3 상태 전이

status == POST 인 경우:
P_t = 1

---

# 9. FCF 정의 및 PV 엔진

## 9.1 FCF (Free Cash Flow) 정의

```
FCF_t = Revenue_t - OPEX_t - Taxes_t - CAPEX_t - ΔWC_t
```

이 때 OPEX, Taxes, CAPEX, ΔWC는 고정 상수비용으로 처리한다.

## 9.2 총비용 구성

```
총비용 = 대출금 원금상환 + 선순위 이자비용 + OPEX + 재무비 + 세금
```

## 9.3 PV 엔진

함수:

PV_t = pv_engine(state, mode)

프로젝트 생존 조건 하에서 미래 운영 현금흐름의 현재가치를 반환한다.

```
V = Σ [ Expected FCF_t / (1 + r)^t ]
```

- 할인율 r = 사회적 할인율 4.5%

입력:
- SMP 윈도우
- REC 윈도우
- 발전량 윈도우
- 할인율
- 출력제한 플래그

출력:
- 스칼라 PV_t

내부 예측 로직은 모듈화되어 교체 가능하다.

# 10. SMP/REC 모델링

## 10.1 가격 과정

SMP와 REC는 GBM(기하브라운운동) 모델을 따른다.

| 항목 | SMP | REC |
|------|-----|-----|
| 초기가격 | 113 원/kWh | 77 원/kWh |
| 드리프트 | +6%/년 | +6%/년 |
| 변동성 | 16%/년 | 14%/년 |
| 하한 | 79 원 | 40 원 |
| 상한 | 159 원 | 134 원 |

**변동성 조정 근거**: 고정가격이 6개월 평균으로 산정되므로, 이론적으로 표본평균의 표준편차 = σ/√6 ≈ 41% 수준으로 감소함. 보수적으로 원래 변동성의 절반(50%)을 적용.

## 10.2 장기계약 구조

편입 당시 T 시점에서 (SMP + 1×REC) 장기계약 체결

- 계약 가격은 편입 시점의 SMP, REC 가격 기준으로 고정

## 10.3 고정가격 산정

프로젝트 편입 시점으로부터 6개월 전의 과거 SMP, REC 데이터 평균으로 추정

# 12. 전이 함수

매 타임스텝에서:

1. 외생 이벤트 샘플링
2. 디폴트 발생 시:
   → status = FAILED
3. PRE이고 t >= 완공월 (고정 12) 일 때:
   → status = POST
4. 다음 월로 상태 갱신

완공 시점에 추가적인 변동성 충격 없음.
인위적인 점프 추가 없음.
체제 전환(regime switch)이 자연스럽게 NAV 구조를 변경함.

# 13. 롤오버 구조 및 토큰 증자

## 13.1 롤오버 필요성

단일 프로젝트는 만기 시 가치가 0이 되는 구조.
롤오버를 통해 영속적인 인프라 투자수단으로 설계.

## 13.2 변수 정의

| 변수 | 설명 |
|------|------|
| a | 기존 STO 토큰 수 |
| b | 기존 STO 토큰의 가격 (= 현재 NAV / a) |
| x | 신규 발행 토큰 수 |
| y | 신규 편입 PF의 **편입 시점 가치** (= 완공후가치 × 초기완공확률, P_COMPLETE_INIT=0.95 적용) |
| Q | 증자하고자 하는 자금조달액 (Q ≤ y) |
| P | 토큰 1개당 매입 비용 (공모가) |

**중요**: y는 완공 후 가치가 아니라 편입 시점의 실제 가치이다. 신규 PF는 아직 건설 중이므로 완공 확률(0.95)을 반영하여 할인한 가치를 사용한다. 이는 NAV 계산 방식과 일치하여 기존/신규 투자자 모두를 보호한다.

## 13.3 토큰 증자 조건 (3가지)

### 조건 1: 기존 주주 보호

```
b ≤ (y + ab) / (a + x)
```

새로운 가격은 기존 가격보다 높거나 같아야 함.

### 조건 2: 자금조달

```
P × x = Q
```

토큰 공모가 × 발행 토큰 수 = 자금조달액

### 조건 3: 신규 주주 보호

```
P ≤ (y + ab) / (a + x)
```

토큰 공모가보다 새로운 가격이 높거나 같아야 함.

## 13.4 예시

조건: a=10, b=9, y=25, Q=20

1. 9 ≤ (90+25)/(신규토큰수+10)
2. 신규토큰당투자금액 ≤ 115/(신규토큰수+10)
3. 신규토큰당투자금액 × 신규토큰수 = 20

결과:
- 토큰 개수 범위: 40/19 < x < 25/9 (약 2.1 ~ 2.8개)
- 토큰 투자금액 범위: 7.2 < P < 9.5
- 시장가격 범위: 9 ~ 9.5

# 14. 낙찰 확률밀도함수

## 14.1 낙찰 경쟁 모델링

현실적 낙찰 경쟁을 반영하기 위한 확률밀도함수 도입.

## 14.2 함수 정의

```
p(x) = a × e^(bx)
```

- 정의역: [9.1, 9.5]
- x = 7.2 (토큰 개수 많음): 낙찰확률 80%
- x = 7.5 (토큰 개수 적음): 낙찰확률 0%

## 14.3 적용

매 프로젝트 편입 시, 위 확률밀도함수에서 낙찰가를 샘플링하여 순현금흐름 현재가치에 반영.
