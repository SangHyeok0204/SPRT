# NAV 시뮬레이션 설계 문서
## 신재생에너지 펀드형 STO (태양광 PF)

---

# 1. 목적

본 문서는 태양광 프로젝트 파이낸싱(PF) 구조 기반 펀드형 STO의 일별 NAV 경로 시뮬레이션을 위한 백본 아키텍처를 정의한다.

시뮬레이션 요구사항:

- 일 단위 타임스텝 사용
- 시뮬레이션 범위:
  - 건설기간 2년 (730일)
  - 운영기간 20년 (7300일)
- NAV 시계열 생성
- 위험중립(Risk-Neutral) / 현실세계(Real-World) 평가 모드 전환 지원
- 희소 외생 포아송 이벤트 반영
- 마르코프 상태 갱신 구조 준수
- 향후 PF 롤오버 편입 확장 가능

본 문서는 최종 가치평가 모델이 아니다.
시뮬레이션 엔진의 백본 구조를 정의하는 문서이다.

---

# 2. 시간 구조

- 타임스텝: 1일
- 건설기간: 730일
- 운영기간: 7300일
- 전체 기간: 8030일

---

# 3. 모델링 프레임워크

마르코프 상태 전이 구조를 사용한다.

매일 t에서:

State S_t → 외생 충격 → 전이 → S_{t+1}

파생 변수 (PV_t, P_complete_t, NAV_t)는 S_t로부터 계산된다.

---

# 4. 상태 변수

상태 S_t에 포함되어야 할 변수:

## 핵심 상태

- `time_t` : 정수 (현재 일 인덱스)
- `status_t` : 범주형
  - "PRE"  : 완공 전
  - "POST" : 완공 후
  - "FAILED" : 건설 디폴트

## 시장 데이터 윈도우

윈도우 크기: n일 (설정 가능)

- `SMP_t` : shape (n, 2)
  - [가격, 거래량]
- `REC_t` : shape (n, 2)
  - [가격, 거래량]
- `energy_amount_t` : shape (n,)

t-n+1일부터 t일까지의 데이터를 나타낸다.

(실제 데이터 및 예측 로직은 추후 PV 엔진에 연결 예정)

## 재무 파라미터

- `r_t` : 고정 사회적 할인율
  - 연 4.5% 고정

## 외생 이벤트 플래그

- `default_today` : bool (당일 디폴트 발생 여부)
- `curtail_today` : bool (당일 출력제한 발생 여부)

---

# 5. 외생 이벤트 (포아송 기반)

일별 베르누이 시행 (포아송 근사)을 가정한다.

## 5.1 건설 디폴트 이벤트

- status == PRE 일 때만 적용
- 일일 확률:
  p_default = 0.001% = 0.00001

발생 시:
→ status = FAILED
→ NAV = 0 (영구)

---

## 5.2 운영 출력제한 이벤트

- status == POST 일 때만 적용
- 일일 확률:
  p_curtail = 0.01% = 0.0001

발생 시:
→ 해당일 SMP 및 REC 수익 = 0
→ PV 계산에 반영

---

# 6. 완공 모델 (위험률 기반)

status == PRE 동안 완공 확률이 매일 갱신된다.

정의:

- λ_t = 일별 위험률 (hazard rate)

갱신 규칙:

P_t = 1 - (1 - P_{t-1}) * exp(-λ_t)

status == POST 인 경우:
P_t = 1

이는 최종 완공에 도달할 누적 확률을 나타낸다.

---

# 7. PV 엔진 (운영 가치)

함수:

PV_t = pv_engine(state, mode)

프로젝트 생존 조건 하에서 미래 운영 현금흐름의 현재가치를 반환한다.

입력:
- SMP 윈도우
- REC 윈도우
- 발전량 윈도우
- 할인율
- 출력제한 플래그

출력:
- 스칼라 PV_t

내부 예측 로직은 모듈화되어 교체 가능하다.

---

# 8. NAV 계산 모드

두 가지 모드를 지원한다:

## 8.1 위험중립 모드 (risk_adjust = "NO")

NAV_t = P_t × PV_t

---

## 8.2 현실세계 모드 (risk_adjust = "YES")

NAV_t = P_t × (요구수익률로 조정된 PV_t)

조정 방법:
- 리스크 프리미엄 승수
또는
- 대안적 할인 방식

백본에서 반드시 분리 구현:

- nav_calculator_rn()
- nav_calculator_rw()

---

# 9. 전이 함수

매 타임스텝에서:

1. 외생 이벤트 샘플링
2. 디폴트 발생 시:
   → status = FAILED
3. PRE이고 t >= 완공일 (고정 730) 일 때:
   → status = POST
4. 다음 날로 상태 갱신

완공 시점에 추가적인 변동성 충격 없음.
인위적인 점프 추가 없음.
체제 전환(regime switch)이 자연스럽게 NAV 구조를 변경함.

---

# 10. 출력

시뮬레이션에서 저장해야 할 항목:

- PV_series (현재가치 시계열)
- P_complete_series (완공확률 시계열)
- NAV_series (순자산가치 시계열)
- Status_series (상태 시계열)

matplotlib를 사용하여 시각화한다.
