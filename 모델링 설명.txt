================================================================================
  태양광 PF 토큰 펀드 NAV 시뮬레이션 — 모델링 설명
================================================================================


1. 개요
--------

본 시뮬레이션에서 미래현금흐름은 태양광 프로젝트가 운영 단계에 진입한 이후
발생하는 전력 판매 수익을 기반으로 구성된다.

태양광 발전사업자의 수익은 크게 두 가지 원천으로 나뉜다.
  - SMP 수익: 전력시장에서 전기를 판매하여 얻는 계통한계가격 수익
  - REC 수익: 신재생에너지 의무할당제 하에서 발전증서를 판매하여 얻는 수익

두 가격 모두 시장에서 변동하는 확률변수이므로, 미래 현금흐름을 확정적인
단일 값으로 예측하는 것은 불가능하다. 따라서 본 모델은 두 가격 각각을
기하 브라운 운동(Geometric Brownian Motion, GBM)으로 모형화하여,
매 타임스텝마다 확률적으로 진화하는 가격 경로를 생성한다.


2. SMP / REC 가격 모형 (GBM)
------------------------------

가격 과정: dS/S = μdt + σdW

  ┌──────────┬──────────────┬──────────────┬──────────────────────────────────┐
  │  항목    │     SMP      │     REC      │  비고                            │
  ├──────────┼──────────────┼──────────────┼──────────────────────────────────┤
  │ 초기가격 │ 113 원/kWh   │ 77 원/kWh    │ 2025년 평균                      │
  │ 드리프트 │ +6%/년       │ +6%/년       │ 장기 +6.8% 반영                  │
  │ 원시변동성│ 32.4%/년     │ 28.1%/년     │ 월별 로그수익률 기반 연환산       │
  │ 적용변동성│ 16%/년       │ 14%/년       │ 6개월 평균 효과로 절반 적용       │
  └──────────┴──────────────┴──────────────┴──────────────────────────────────┘

SMP:
  - 초기가격은 2025년 EPSIS 월별 가중평균 SMP의 연평균(113원/kWh)
  - 드리프트: 과거 장기 평균 변화율 +6.8%를 반영하여 +6% 적용
  - 원시 변동성: 2001~2025년 297개월 월별 로그수익률 표준편차 × √12 = 32.4%

REC:
  - 초기가격은 현물시장 최근 평균가 77,000원/REC → kWh 환산 77원/kWh
  - 원시 변동성: 결측치 선형보간 후 산정한 28.1%
  - 드리프트: 장기 추세를 반영하여 SMP와 동일하게 +6% 적용

변동성 조정:
  장기계약 고정가격이 편입 시점 직전 6개월의 평균 (SMP + REC)으로
  산정되므로, 표본평균의 변동성은 이론적으로 원시 변동성의
  1/√6 ≈ 0.41배로 축소된다. 보수적으로 원시 변동성의 절반(50%)을
  적용하여, SMP는 32.4% → 16%, REC는 28.1% → 14%로 설정하였다.


3. SMP-REC 상관관계
--------------------

SMP와 REC의 월별 로그수익률 간 피어슨 상관계수는 ρ = -0.03으로,
통계적으로 유의하지 않은 수준이다. 사실상 두 가격이 독립적으로 움직인다고
해석할 수 있다.

다만 모델의 구조적 확장성을 위해 상관 브라운 운동 프레임워크는 유지하였다.

  Z_smp  ← 독립 표준정규
  Z_indep ← 독립 표준정규
  Z_rec  = ρ · Z_smp + √(1 - ρ²) · Z_indep


4. 발전량 모형
--------------

태양광 발전은 일사량과 기온에 강하게 의존하기 때문에 뚜렷한 계절성을 보인다.

(1) 계절계수 산정

한국전력거래소가 제공하는 2024년 일별 태양광 발전량 실데이터로부터
월별 계절계수를 산정하였다.

  계절계수 = 해당 월 일평균 발전량 / 연간 일평균 발전량

  - 최고: 5월 (1.417)
  - 최저: 2월 (0.657)
  - 7월은 장마 영향으로 0.954까지 하락 → 8월에 1.256으로 회복

(2) 계절 진폭 축소 (Dampening)

원시 계절계수는 전국 태양광 발전 합계 데이터에서 산출된 값이므로,
개별 1MW 프로젝트의 계절 패턴과 괴리가 존재한다.

  - 전국 합계에는 수평면 옥상형, 비최적 경사각 시스템 등이 혼재
  - 최적 경사각(약 30도)으로 설계된 단일 지상형 프로젝트는
    겨울 발전량이 상대적으로 높고, 여름 발전량이 상대적으로 낮아
    계절 진폭이 축소됨

평균 회귀 축소 변환을 적용:

  S'_m = 1 + α × (S_m - 1),   α = 0.1

  ┌────────────────────┬────────┬────────┐
  │ 지표               │ 축소 전│ 축소 후│
  ├────────────────────┼────────┼────────┤
  │ 변동계수 (CV)      │  0.24  │  0.024 │
  │ 피크/트러프 비율   │  2.16  │  1.08  │
  └────────────────────┴────────┴────────┘

  α = 0.1은 원시 계절 진폭의 10%만 유지하는 것으로,
  단일 최적경사각 프로젝트의 실질 계절 변동과 정합적인 수준이다.

(3) 발전량 생성

  월 발전량 = 기저 발전량 × 축소 계절계수 × (1 + 탈계절화 변동성(0.005) × Z)

  - Z: 표준정규 노이즈
  - 발전량은 물리적으로 음수 불가 → max(0, ...) 클리핑 처리


5. 월별 매출액
--------------

  월 매출액 = (SMP 가격 + REC 가격) × 발전량(MWh) × 1,000

  - × 1,000: 가격 단위(원/kWh)와 발전량 단위(MWh)를 kWh로 통일하기 위한 환산
  - SMP와 REC를 동일한 원/kWh 단위로 통일하여 단순 합산 가능하도록 설계


6. 비용 구조 및 FCF
---------------------

매출액에서 비용을 차감하여 세전 현금흐름(Gross CF)을 산출한다.

(1) 비용 항목

  ┌──────────────────────┬──────────────┬─────────────┬──────────────────────┐
  │ 항목                 │ 연간         │ 월간        │ 적용 기간            │
  ├──────────────────────┼──────────────┼─────────────┼──────────────────────┤
  │ 운영비 (OPEX)        │ 2,000만 원   │ 167만 원    │ 운영 전 기간 (20년)  │
  │ 사무위탁비           │ 1,500만 원   │ 125만 원    │ 운영 전 기간 (20년)  │
  │ 대수선비             │   500만 원   │  42만 원    │ 운영 전 기간 (20년)  │
  │ (고정비용 소계)      │ 4,000만 원   │ 333만 원    │                      │
  ├──────────────────────┼──────────────┼─────────────┼──────────────────────┤
  │ 선순위 대출 이자     │ 2,400만 원   │ 200만 원    │ 운영 1~15년차만      │
  │ 대출 원금상환        │ 2,667만 원   │ 222만 원    │ 운영 1~15년차만      │
  ├──────────────────────┼──────────────┼─────────────┼──────────────────────┤
  │ 월 총비용 (1~15년)   │              │ 756만 원    │                      │
  │ 월 총비용 (16~20년)  │              │ 333만 원    │ 56% 감소             │
  └──────────────────────┴──────────────┴─────────────┴──────────────────────┘

  - 선순위 대출: 4억 원, 연 6% 고정금리, 15년(180개월) 균등 원금상환
  - 이자와 원금상환은 상환기간(15년) 동안만 발생
  - 운영 181개월 이후에는 이자·원금상환이 동시에 종료 → 고정비용만 잔존

(2) FCF 산정

  Gross CF = 매출액 - 고정비용 - 이자 - 원금상환

  if Gross CF > 0:
      세후 FCF = Gross CF × (1 - 0.22)       ← 법인세 22%
  else:
      세후 FCF = Gross CF                     ← 결손금 처리 (세금 없음)

(3) 수익 분배

  세후 FCF는 지분투자자와 토큰투자자에게 1:3 비율로 분배된다.

  - 지분투자자 (Equity): 25%
  - 토큰투자자 (Token):  75%  ← 분배금의 기초


7. 태양광 패널 열화 (Degradation)
----------------------------------

태양광 모듈은 자외선 노출, 온도 순환, 습기 침투 등에 의해 시간이 지남에 따라
출력이 점진적으로 감소한다. 업계 표준 보증 기준으로 연간 약 0.5%의 열화율이
적용된다.

  열화 계수 = (1 - 0.005) ^ (운영 경과 월수 / 12)

  ┌──────────────┬──────────┐
  │ 운영 경과    │ 열화계수 │
  ├──────────────┼──────────┤
  │ 건설 중(PRE) │  1.000   │
  │ 10년차       │  0.951   │
  │ 20년차       │  0.905   │
  └──────────────┴──────────┘

  - 건설 중(PRE 상태)에는 패널 미가동 → 열화 발생 않음 (계수 = 1.0)
  - 월별 매출액에 직접 곱해져 반영:
    monthly_revenue = price × avg_energy × 1000 × degradation_factor
  - 분배금 산출에도 동일하게 적용


8. 현재가치(PV) 엔진
---------------------

PV 엔진은 월별 현금흐름 추정치를 미래 잔여 운영기간 전체에 걸쳐 할인하여
단일 스칼라 값으로 응축하는 역할을 한다.

(1) 할인율

  - 연 할인율: 4.5% (사회적 할인율)
  - 월 할인율: r = (1.045)^(1/12) - 1 ≈ 0.3675%

(2) 감소 연금 (Declining Annuity)

PV 산출 시 열화에 의한 발전량 감소를 반영해야 하므로,
단순 연금현가계수 대신 감소 연금을 사용한다.

  g_m = 월별 열화율 = 1 - (1 - 0.005)^(1/12) ≈ 0.0004175
  q   = (1 - g_m) / (1 + r)

  감소 연금현가계수 = q × (1 - q^N) / (1 - q)

여기에 현재 시점까지의 누적 열화도(degradation_factor)를 곱하여,
이미 진행된 열화가 미래 현금흐름의 출발점에 반영되도록 하였다.

(3) 프로젝트 상태별 PV 산출

  PRE (건설 중):
    - 열화 없음 (degradation_factor = 1.0)
    - 잔여 건설기간만큼 추가 할인 적용
    - 운영기간 전체 240개월에 대한 감소 연금현가계수 적용

  POST (운영 중):
    - 잔여 운영기간에 대한 감소 연금현가계수로 PV 산출
    - 건설 지연에 따른 추가 할인 없음

  신규 PF (롤오버):
    - 동일한 감소 연금을 적용하여 보수적 가치 평가

  FAILED (실패):
    - 미래현금흐름 없음 → PV = 0


9. 시장 데이터 윈도우
----------------------

PV 엔진이 미래현금흐름을 추정할 때 사용하는 정보는
가장 최근 6개월의 시장 데이터 윈도우이다.

  - SMP 가격, REC 가격, 발전량 각각에 대해 최근 6개 관측치를
    슬라이딩 윈도우 방식으로 유지
  - 윈도우 평균값을 "향후에도 이 수준이 지속될 것"이라는 단순 예측치로 사용
  - 현재 단계의 플레이스홀더 설계이며,
    향후 보다 정교한 예측 모델로 교체 가능하도록 PV 엔진은 모듈화되어
    전이 함수로부터 분리되어 있음


10. 출력제한 (Curtailment)
---------------------------

출력제한은 전력 계통의 수급 상황에 따라 발전소가 강제로 출력을
줄여야 하는 사건이다.

  - 발생 확률: 월별 0.3% (베르누이 시행)
  - 발생 시: 해당 월의 SMP/REC 수익 = 0
  - PV 반영: 해당 월의 현금흐름 한 달분을 차감


11. NAV 산출
-------------

PV 값은 완공 확률 P_t와 결합되어 NAV를 구성한다.

  위험중립 평가:  NAV_t = P_t × PV_t
  현실세계 평가:  NAV_t = P_t × PV_t × (1 - π)     ← 리스크 프리미엄 차감

  건설기:
    - P_t는 0.7에서 시작 → hazard rate에 의해 점진적으로 1에 수렴
    - NAV는 완공 확률만큼 할인된 값

  운영기 전환 후:
    - P_t = 1.0 → NAV = PV
    - 시장 가격 변동과 잔여 운영기간 감소에 의해서만 NAV 결정
    - 가격이 일정하다면 PV는 시간에 따라 자연스럽게 감소


12. 종합
---------

본 모델의 미래현금흐름 반영 체계:

  실데이터 기반 확률 과정으로 시장 가격 생성
    → 계절성 반영 발전량과 결합하여 월별 현금흐름 구성
      → DCF 방식으로 잔여 운영기간 전체를 할인하여 현재가치 산출
        → 완공 확률에 의해 가중 → 최종 NAV

건설기의 불확실성과 운영기의 시장 변동성이 하나의 일관된 프레임워크 안에서
통합적으로 반영된다.


================================================================================
  토큰 병합(Reverse Split) 메커니즘
================================================================================

1. 배경
--------

롤오버가 반복되면 신규 PF 편입을 위한 토큰이 누적 발행되어 총 토큰 수가
지속적으로 증가한다. 기존 PF의 운영기간 만료에 따른 NAV 감소와 맞물리면
토큰당 가격이 장기적으로 하락하는 구조적 문제가 발생한다.
이를 방치하면 토큰 가격이 0에 수렴하여 투자자 가치가 사실상 소멸될 수 있다.


2. 원리
--------

토큰 병합은 펀드의 총 NAV(시가총액)를 변동시키지 않으면서
유통 토큰 수를 줄여 개당 가격을 복원하는 메커니즘이다.

  - 주식시장의 액면병합(주식병합)과 동일한 원리
  - 투자자의 총 보유가치는 불변, 보유 단위 수가 줄고 단위당 가격 상승
  - 예: 2:1 병합 시 토큰 2개 → 1개, 개당 가격 2배


3. 설계
--------

  (1) 병합 기준가
      시뮬레이션 20년차(Month 240) 롤오버 직후의 토큰 가격으로 설정.
      초기 PF가 모두 퇴출된 후 순수 롤오버 기반으로 전환되는 전환점이므로,
      장기 균형 가격의 기준으로 적합하다.

  (2) 트리거 조건
      매월 토큰 가격(NAV/토큰수)이 기준가의 50% 이하로 하락하면 발동.

  (3) 병합 비율
      2:1 — 토큰 수를 절반으로 줄여 개당 가격을 2배로 복원.


4. 피드백 효과
--------------

토큰 증자 공식에서 신규 발행 가능 토큰의 상한은 x_max = y / b로,
현재 토큰 가격 b에 반비례한다.

  병합으로 b 상승 → x_max 감소 → 다음 롤오버에서 과도한 토큰 발행 억제

이는 펀드 가치의 장기적 안정성에 기여하는 자기조절 메커니즘으로 작동한다.
