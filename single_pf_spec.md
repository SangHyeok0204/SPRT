## 3. 수익 산정

### 3-1. 고정가격 결정

```
고정가격 = avg(SMP_buffer) + avg(REC_buffer)
```

초기가격 기준: 113 + 77 = **190 원/kWh**

### 3-2. 월 매출액

```
월 매출액 = 고정가격 × 월 평균 발전량 × 1,000 × 열화계수

| 월 평균 발전량 | 105.9 | MWh |
| 단위 환산 | ×1,000 | kWh/MWh |
| **월 매출액 (열화 전)** | **20,121,000** | **원** |

### 3-3. 열화 (Degradation)

태양광 패널은 연 0.5%씩 출력이 감소한다.

```
degradation_factor = (1 - 0.005) ^ (운영경과년수)

## 4. 비용 구조

본 모델에서 월별 비용은 세 항목으로 구성된다: 운영비(OPEX), 선순위 대출 이자, 대출 원금상환.
건설기(PRE)에는 비용이 발생하지 않으며, **운영 개시(POST) 이후**부터 차감된다.

### 4-1. OPEX (운영비)

```
OPEX_monthly = OPEX_annual / 12
             = 20,000,000 / 12
             = 1,666,667 원/월
```

1MW 태양광 발전소의 연간 운영유지비를 **2천만 원**으로 가정하고, 12로 나누어 월액을 산출한다.
운영 전 기간(20년) 동안 **고정 비용**으로 매월 동일 금액이 차감된다.

### 4-2. 선순위 대출 이자

```
monthly_interest = SENIOR_DEBT × SENIOR_INTEREST_RATE / 12
                 = 400,000,000 × 0.05 / 12
                 = 1,666,667 원/월
```

선순위 대출 4억 원에 대해 **연 5% 고정금리**를 적용한다.
본 모델에서는 단순화를 위해 잔액 기준이 아닌 **원금 기준 고정 이자**로 처리한다.
이자는 **대출 상환기간(15년) 동안만** 발생하며, 원금 상환이 완료되면 이자도 함께 종료된다.

### 4-3. 대출 원금상환

```
DEBT_MONTHLY_PAYMENT = SENIOR_DEBT / (DEBT_REPAYMENT_YEARS × 12)
                     = 400,000,000 / (15 × 12)
                     = 400,000,000 / 180
                     = 2,222,222 원/월
```

4억 원을 **15년(180개월) 균등 원금상환** 방식으로 갚는다.
매월 동일한 금액(약 222만 원)을 상환하며, **운영 180개월차(=15년)에 상환이 완료**된다.

상환 완료 시점을 PF 수명 기준으로 환산하면:
- 건설기 12개월 + 운영 180개월 = **PF local_time 192개월 (16년차)**
- 운영 181개월~240개월 (16~20년차, 약 5년간)은 원금상환 = 0

코드에서의 분기 로직:

```python
op_months = max(0, pf.local_time - T_CONSTRUCTION)   # 운영 경과 월수
if op_months < DEBT_REPAYMENT_YEARS * 12:             # < 180
    monthly_interest = SENIOR_DEBT * (SENIOR_INTEREST_RATE / 12)  # 1,666,667원
    monthly_debt = DEBT_MONTHLY_PAYMENT               # 2,222,222원
else:
    monthly_interest = 0.0                             # 상환 완료 → 이자도 종료
    monthly_debt = 0.0                                 # 상환 완료
```

### 4-4. 월별 총비용 요약

| 항목 | 운영 1~15년차 | 운영 16~20년차 | 산출식 |
|------|-------------:|---------------:|--------|
| OPEX | 1,666,667원 | 1,666,667원 | 20,000,000 / 12 |
| 선순위 이자 | 1,666,667원 | **0원** | 400,000,000 × 0.05 / 12 → 상환 완료 시 종료 |
| 대출 원금상환 | 2,222,222원 | **0원** | 400,000,000 / 180 → 180개월 후 종료 |
| **월 총비용** | **5,555,556원** | **1,666,667원** | |

15년차 → 16년차 전환 시 이자와 원금상환이 동시에 종료되면서 **월 비용이 70% 감소**한다.

## 5. 월별 FCF (Free Cash Flow) 산정

### 5-1. 산정 공식

FCF는 매출액에서 모든 비용을 차감한 후 세금을 적용하여 산출한다.

**Step 1: Gross CF (세전 현금흐름)**

```
Gross CF = 월 매출액 - OPEX_monthly - monthly_interest - monthly_debt
```

각 항목을 풀어 쓰면:

```
Gross CF = (고정가격 × 발전량 × 1000 × 열화계수)
         - (20,000,000 / 12)
         - (400,000,000 × 0.05 / 12)   ← 15년 이내일 때만
         - (400,000,000 / 180)          ← 15년 이내일 때만
```

**Step 2: 세후 FCF**

```
if Gross CF > 0:
    세후 FCF = Gross CF × (1 - TAX_RATE)     # 법인세 22% 차감
              = Gross CF × 0.78
else:
    세후 FCF = Gross CF                       # 손실 시 세금 없음 (결손금 처리)
```

**Step 3: 토큰 귀속분**

```
token_cf = 세후 FCF × TOKEN_SHARE
         = 세후 FCF × 0.75
```

전체를 하나의 식으로 정리하면 (Gross CF > 0인 경우):

```
token_cf = (매출액 - OPEX - 이자 - 원금상환) × 0.78 × 0.75
```

### 5-2. 운영 시기별 FCF (초기가격 기준, 열화 반영)

| 시기 | 월 매출 | 월 총비용 | Gross CF | 세후 FCF |
|------|--------:|----------:|---------:|---------:|
| **1년차** (열화 0%) | 20,121,000 | 5,555,556 | 14,565,444 | **11,361,047** |
| **10년차** (열화 4.9%) | 19,233,454 | 5,555,556 | 13,677,899 | **10,668,761** |
| **16년차** (상환 완료, 열화 7.2%) | 18,663,615 | 1,666,667 | 16,996,949 | **13,257,620** |
| **20년차** (열화 9.5%) | 18,293,133 | 1,666,667 | 16,626,466 | **12,968,644** |

16년차에 FCF가 1년차보다 높은 이유: 열화로 매출은 7.2% 감소하지만, 대출 원금상환(월 222만 원)과 이자(월 167만 원)가 동시에 종료되어 비용이 70% 감소하기 때문이다.

## 6. 분배금 산정 로직

### 6-1. 수익 분배 구조

세후 FCF는 지분투자자와 토큰투자자에게 **1:3 비율**로 분배된다.

| 귀속 대상 | 비율 | 산출식 |
|-----------|-----:|--------|
| 지분투자자 (Equity) | 25% | 세후 FCF × 0.25 |
| **토큰투자자 (Token)** | **75%** | **세후 FCF × 0.75** |

### 6-2. 분배 조건

분배금이 발생하려면 다음 조건을 **모두** 충족해야 한다:

1. PF 상태가 **POST** (운영 중)일 것 — 건설 중(PRE), 실패(FAILED)는 분배 없음
2. 해당 월에 **출력제한(curtailment)이 없을 것** — 발생 시 해당 월 매출 = 0
3. 운영기간이 **종료되지 않았을 것** — local_time < 252
4. 토큰 분배금이 **양수(+)일 것** — 음수 분배 없음 (`max(0, ...)` 처리)

### 6-3. 분배금 산정 절차 (코드 기준)

```
① price ← 장기계약 고정가격 (완공 시 확정된 SMP+REC 6개월 평균)
② avg_energy ← 최근 6개월 발전량 평균
③ degradation_factor ← (1 - 0.005) ^ (운영경과월수 / 12)
④ monthly_revenue ← price × avg_energy × 1000 × degradation_factor
⑤ gross_cf ← monthly_revenue - OPEX - 이자 - 원금상환  (이자·원금은 상환기간 내만)
⑥ monthly_fcf ← gross_cf × (1 - 0.22)   (gross_cf > 0)
⑦ token_distribution ← monthly_fcf × 0.75
⑧ 분배금 ← max(0, token_distribution)
```

### 6-4. 토큰당 분배금 (초기가격 기준)

| 시기 | PF당 토큰분배 | 토큰당 월 분배금 | 토큰당 연 분배금 |
|------|-------------:|-----------------:|-----------------:|
| **1년차** | 8,520,785원 | **71.0원** | **852원** |
| **10년차** | 8,001,571원 | **66.7원** | **800원** |
| **16년차** (상환 완료) | 9,943,215원 | **82.9원** | **994원** |
| **20년차** | 9,726,483원 | **81.1원** | **973원** |

- 토큰 발행가(10,000원) 대비 **표면 수익률**: 약 **8.5%** (1년차 기준), **9.9%** (16년차 이후)
- 15년 상환 완료 후 이자+원금상환 부담이 동시에 사라지면서 분배금이 약 40% 증가

## 7. 미래현금흐름 현가 (PV)

### 7-1. 할인율

| 항목 | 값 |
|------|---:|
| 연 할인율 | 4.5% |
| 월 할인율 | 0.3675% |

### 7-2. 감소 연금 (Declining Annuity)

열화에 의해 매월 현금흐름이 체감하므로, 단순 연금현가계수 대신 **감소 연금**을 사용한다.

```
g_m = 월별 열화율 = 1 - (1 - 0.005)^(1/12) ≈ 0.0004175
r   = 월별 할인율 = (1.045)^(1/12) - 1      ≈ 0.003675

q = (1 - g_m) / (1 + r)

감소 연금현가계수 = q × (1 - q^N) / (1 - q)
```

| 방식 | Annuity Factor (N=240) | PV (토큰분) |
|------|-----------------------:|------------:|
| 단순 연금 (열화 무시) | 159.2891 | 1,357,268,309원 |
| **감소 연금 (열화 반영)** | **152.6350** | **1,300,570,193원** |
| **열화에 의한 PV 감소** | | **-4.18%** |

### 7-3. PF 상태별 PV 산출

#### POST 상태 (운영 중)

```
PV = token_cf × annuity(remaining_op) × degradation_factor
```

- `token_cf`: 당월 기준 토큰 현금흐름 (열화 반영된 시점의 값)
- `annuity(N)`: 잔여 운영기간 N에 대한 감소 연금현가계수
- `degradation_factor`: 현재까지 누적 열화도 (미래 CF의 출발점)
- 출력제한 발생 시: `PV -= token_cf` (해당 월분 차감)

#### PRE 상태 (건설 중)

```
PV = token_cf × annuity(T_OPERATION) × (1+r)^(-months_to_start)
NAV = P_complete × PV
```

- 건설기에는 열화 없음 (degradation_factor = 1.0)
- 잔여 건설기간만큼 추가 할인 적용
- 완공 확률 P_t를 곱하여 NAV 산출

### 7-4. 완공 시점 PV 요약 (편입 기준)

| 단계 | 금액 | 비고 |
|------|-----:|------|
| 완공 후 PV (토큰분, 열화 반영) | 1,300,570,193원 | 감소 연금 적용 |
| × 건설기 할인 (12개월) | 1,244,564,778원 | × 0.9569 |
| × 완공 확률 (P=0.95) | **1,182,336,539원** | **NAV 반영값** |

## 9. 신규 PF 가치 추정 (롤오버 시)

롤오버로 신규 PF를 편입할 때, 해당 시점의 시장 가격으로 가치를 추정한다.

```
① avg_smp, avg_rec ← 현재 SMP/REC 버퍼 6개월 평균
② monthly_revenue ← (avg_smp + avg_rec) × avg_energy × 1000
③ gross_cf → 세후 FCF → token_cf (동일 로직)
④ annuity ← 감소 연금 (N = T_OPERATION = 240)
⑤ discount ← (1+r)^(-12)  (건설기 할인)
⑥ single_pf_value = token_cf × annuity × discount
⑦ new_pf_value = single_pf_value × ROLLOVER_PF_COUNT
```

- 신규 PF는 아직 운영 전이므로 **degradation_factor = 1.0**
- 그러나 미래 PV 계산에는 **감소 연금**을 적용하여 열화 반영
- NAV에는 `new_pf_value × P_COMPLETE_INIT(0.95)`로 반영
